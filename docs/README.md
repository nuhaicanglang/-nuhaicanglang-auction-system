# 在线拍卖系统(Online Auction System)

> Java Web 高级开发技术大作业 · 企业级实战项目
>
> 目标:以"工程化+教学化"的方式,从 0 到 1 设计、开发、部署一个具备**高并发竞拍**能力的在线拍卖系统,系统性掌握 Java 后端核心技术栈。

---

## 一、项目定位

不是一个"能跑起来的 CRUD"项目,而是一个**贴近生产环境**的在线拍卖系统:

- 真实业务场景:出价并发、超时结算、支付、订单、通知。
- 真实技术挑战:高并发安全(防止超出价/重复出价/丢失出价)、实时推送、异步解耦、全文检索、缓存一致性。
- 真实工程规范:分层架构、统一返回、全局异常、参数校验、API 文档、单元测试、CI、容器化部署。

## 二、技术栈一览

| 层次 | 技术 | 主要职责 |
|------|------|---------|
| 前端 | Vue 3 + Vite + Pinia + Vue Router + Element Plus + Axios | SPA、状态管理、路由、组件库 |
| 网关/反代 | Nginx | 静态资源、反向代理、负载均衡、HTTPS |
| 后端框架 | Spring Boot 3.x、Spring MVC、Spring Validation | Web 容器、参数校验 |
| 安全 | Spring Security + JWT + BCrypt | 认证、授权、密码加密 |
| ORM | MyBatis-Plus | 持久层、分页、代码生成 |
| 数据库 | MySQL 8.x | 主存储、事务 |
| 缓存 | Redis 7.x | 缓存、分布式锁、排行榜、计数器、限流 |
| 消息队列 | RabbitMQ(主),Kafka(备选对比) | 异步解耦、削峰、最终一致性 |
| 搜索 | Elasticsearch 8.x + IK 分词器 | 商品全文检索、聚合 |
| 实时通信 | WebSocket(STOMP) | 实时出价推送 |
| 定时任务 | XXL-Job 或 Spring Scheduling + Redisson | 拍卖到期结算 |
| 工具 | Lombok、MapStruct、Hutool、Knife4j(Swagger) | 简化开发、API 文档 |
| 测试 | JUnit5、Mockito、JMeter | 单测、压测 |
| 部署 | Docker + docker-compose + Nginx | 容器化、一键启动 |
| 监控(可选) | Spring Boot Actuator + Prometheus + Grafana | 健康检查、指标可视化 |

## 四、推荐学习节奏

| 阶段 | 时间 | 产出 | 你将学到 |
|------|------|------|----------|
| 0 设计 | 2~3 天 | docs/ 全部文档 | 需求分析、架构思维、技术选型方法论 |
| 1 骨架 | 1~2 天 | 后端多模块 + 前端工程 + 数据库 | 工程化、Maven、Vite、规范 |
| 2 用户 | 2~3 天 | 注册/登录/JWT/权限/AOP 日志 | 认证授权、Spring Security、AOP |
| 3 商品 | 2 天 | 商品/分类 CRUD、上传图片 | MyBatis-Plus、文件存储 |
| 4 拍卖核心 | 4~5 天 | 出价、Redis 锁、Lua、WS 推送 | 高并发、分布式锁、原子操作、WebSocket |
| 5 异步 | 2~3 天 | RabbitMQ:通知/订单/到期 | MQ 模型、最终一致性、死信队列 |
| 6 搜索 | 2 天 | ES 商品检索、聚合、同步 | 全文检索、分词、数据同步 |
| 7 完善 | 2~3 天 | 订单/支付/评价/统计 | 业务闭环、报表 |
| 8 上线 | 2 天 | Docker、Nginx、压测、报告 | 容器化、部署、性能调优 |

> 总计约 3~4 周的高强度学习。每个阶段我都会先讲"为什么这么做",再带你写代码,最后让你独立扩展。

## 五、如何使用本仓库学习

1. **先读 `docs/`**:不要急着写代码,把设计文档读懂、改成你自己的话术。
2. **逐模块开发**:每个阶段结束都能跑通、能演示。
3. **提问驱动**:每个技术点都问自己三个问题:
   - 这个技术解决了什么问题?
   - 不用它会发生什么?
   - 还有哪些替代方案,为什么选它?
4. **动手压测**:核心出价接口必须用 JMeter 压测,亲眼看到"超出价"问题再亲手解决,印象最深。
