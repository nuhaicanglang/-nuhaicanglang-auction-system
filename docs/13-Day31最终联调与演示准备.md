## 3. 答辩演示流程建议

如需现场演示或后续补录，可按“业务价值 → 架构亮点 → 核心流程 → 性能与部署”展开；本次收尾不要求强制录制视频。

### 3.1 片头：项目概览（约 40 秒）

讲解点：

- 项目名称：在线拍卖系统。
- 技术栈：Spring Boot 3、MyBatis-Plus、Spring Security + JWT、Redis、RabbitMQ、Elasticsearch、Docker Compose。
- 核心功能：用户认证、商品发布审核、实时出价、一口价、反狙击、订单支付、钱包账本、信用分、搜索、管理统计、Excel 导出。

### 3.2 用户主流程演示（约 2 分钟）

建议步骤：

1. 调用 `/api/system/users/login` 登录。
2. 调用 `/api/categories/tree` 查看分类。
3. 调用 `/api/items` 查看商品列表。
4. 调用 `/api/search/items` 搜索“国画”。
5. 调用 `/api/items/{itemId}/bids` 出价，展示 `X-Idempotent-Key`。
6. 调用 `/api/orders` 查看成交后订单。
7. 调用 `/api/orders/{id}/pay` 模拟支付。
8. 调用 `/api/orders/{id}/complete` 完成订单。
9. 调用评价/信用分接口，展示信用变更。

### 3.3 高并发出价设计（约 1 分钟）

讲解点：

- 出价不是直接写 MySQL，而是先走 Redis Lua。
- Lua 保证价格比较、幂等、状态判断是原子操作。
- MQ 异步落库削峰，`client_request_id` 唯一索引保证消费幂等。
- Day28 压测脚本覆盖 `stress-bid.ps1`、`stress-list.ps1`、`stress-search.ps1`。

### 3.4 搜索与同步（约 1 分钟）

讲解点：

- 商品同步到 Elasticsearch 的 `auction_items` 索引。
- 搜索接口支持关键词、状态、分类、价格、排序、高亮、分面聚合。
- 商品审核、下架、成交、出价价格变化通过 MQ 同步 ES。

### 3.5 管理端能力（约 1 分钟）

讲解点：

- 商品审核和强制下架。
- 钱包调账和流水审计。
- 信用分人工调整和信用日志。
- `/api/admin/stats/*` 管理仪表盘统计。
- `/api/admin/export/*` EasyExcel 导出订单、用户、钱包流水。

### 3.6 部署与运维（约 1 分钟）

讲解点：

- `auction-backend/Dockerfile` 多阶段构建。
- `deploy/docker-compose.yml` 编排 MySQL、Redis、RabbitMQ、ES、Kibana、MinIO、Backend、Nginx。
- `application-prod.yml` 使用环境变量连接容器服务名。
- `test/ops/day30-deploy-check.ps1` 可做部署前静态校验。

### 3.7 收尾：项目亮点（约 40 秒）

建议强调：

- 高并发出价的 Redis Lua 原子性。
- MQ 异步削峰和幂等消费。
- 钱包账本的资金审计思路。
- 信用分体系和定时恢复。
- ES 搜索体验。
- Docker 化部署和故障演练文档。
