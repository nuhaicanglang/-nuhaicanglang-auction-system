## 2. 后端基础信息

### 2.1 开发环境地址

```text
Base URL: http://localhost:8080
API 前缀: /api
```

### 2.2 统一响应格式

后端接口统一返回：

```json
{
  "code": 0,
  "msg": "success",
  "data": {}
}
```

前端约定：

- `code === 0`：业务成功，读取 `data`。
- `code !== 0`：业务失败，弹出 `msg`。
- HTTP `401` 或业务未登录错误：清理本地 token，跳转登录页。
- HTTP `403`：提示“无权限访问”。

### 2.3 认证 Header

除公开接口外，均需要：

```http
Authorization: Bearer <accessToken>
```

## 3. 登录与静默刷新 Token

### 3.1 登录

```http
POST /api/system/users/login
Content-Type: application/json
```

请求体：

```json
{
  "username": "admin",
  "password": "123456",
  "captchaId": "可选",
  "captchaCode": "可选"
}
```

成功后保存：

- `token`：Access Token，用于 `Authorization`。
- `refreshToken`：刷新 Token，用于静默续期。
- `user`：当前用户基础信息。
- `roles`：角色编码列表，用于菜单和路由权限。

### 3.2 刷新 Token

```http
POST /api/system/users/refresh
Content-Type: application/json
```

前端建议策略：

- 每次请求收到 `401` 时，先尝试使用 `refreshToken` 刷新。
- 同一时间只允许一个刷新请求，其他失败请求排队等待。
- 刷新成功后重放原请求。
- 刷新失败则清理本地认证状态并跳转登录页。

伪代码：

```ts
let refreshing = false
let waitQueue: Array<(token: string) => void> = []

async function handle401(originalRequest) {
  if (!refreshing) {
    refreshing = true
    try {
      const newToken = await refreshAccessToken()
      waitQueue.forEach(fn => fn(newToken))
      waitQueue = []
      return retry(originalRequest, newToken)
    } catch (e) {
      clearAuth()
      redirectToLogin()
      throw e
    } finally {
      refreshing = false
    }
  }

  return new Promise(resolve => {
    waitQueue.push(token => resolve(retry(originalRequest, token)))
  })
}
```

## 5. 前端请求封装建议

### 5.1 请求拦截器

请求发出前：

- 从本地状态读取 `token`。
- 自动设置 `Authorization: Bearer xxx`。
- 对写操作生成并携带 `X-Idempotent-Key`，例如出价/支付/调账。
- 打开全局 loading 计数器。

### 5.2 响应拦截器

响应返回后：

- 关闭 loading 计数器。
- `code === 0` 直接返回 `data`。
- `code !== 0` 弹出错误提示。
- `401` 进入静默刷新流程。
- `403` 弹出无权限提示。

### 5.3 下载接口处理

Excel 导出接口不要按 JSON 解析，应使用 blob：

```ts
async function downloadExcel(url: string, filename: string) {
  const res = await request.get(url, { responseType: 'blob' })
  const blob = new Blob([res], {
    type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet'
  })
  const link = document.createElement('a')
  link.href = URL.createObjectURL(blob)
  link.download = filename
  link.click()
  URL.revokeObjectURL(link.href)
}
```

## 6. 路由守卫与权限菜单

### 6.1 路由 meta 建议

```ts
{
  path: '/admin/stats',
  component: AdminStatsPage,
  meta: {
    requiresAuth: true,
    roles: ['ADMIN', 'SUPER_ADMIN'],
    title: '统计仪表盘'
  }
}
```

### 6.2 守卫规则

- `requiresAuth=false`：匿名可访问。
- `requiresAuth=true` 且无 token：跳转 `/login?redirect=当前路径`。
- 有 token 但无用户信息：调用 `/api/system/users/me` 或使用登录缓存恢复。
- `roles` 非空时：当前用户角色必须命中任意一个，否则跳转 `403`。

### 6.3 菜单过滤

菜单与路由共用同一份 `roles` 元数据，避免“菜单能看到但路由进不去”。

## 7. 全局加载与错误提示

### 7.1 Loading

使用计数器而不是布尔值：

- 请求开始：`loadingCount++`
- 请求结束：`loadingCount--`
- `loadingCount > 0` 时显示全屏/顶部进度条

这样并发请求不会提前关闭 loading。

### 7.2 错误提示

建议错误分级：

- 表单校验错误：显示在表单字段下方。
- 业务错误：toast/message 弹出。
- 权限错误：跳转 403 页面。
- 登录过期：自动刷新，刷新失败才提示重新登录。
- 网络错误：提示“网络异常，请稍后重试”。

## 9. 常见问题排查

| 问题 | 可能原因 | 处理 |
|---|---|---|
| 401 未登录 | token 缺失/过期 | 检查 Authorization，触发刷新 token |
| 403 无权限 | 当前账号不是管理员 | 换 ADMIN/SUPER_ADMIN 账号 |
| Excel 下载乱码 | 前端把 blob 当 JSON 解析 | 设置 `responseType: 'blob'` |
| 出价重复 | 前端重复点击/无幂等 key | 禁用按钮并传 `X-Idempotent-Key` |
| 搜索无数据 | ES 未启动/未同步 | 启动 ES，检查 Day 24 同步日志 |
| 图表为空 | 数据库暂无订单/支付数据 | 跑一遍完整成交支付流程 |
