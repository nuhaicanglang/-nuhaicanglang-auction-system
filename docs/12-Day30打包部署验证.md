## 2. 后端 Dockerfile 设计

后端镜像使用多阶段构建：

1. `maven:3.9.6-eclipse-temurin-17` 构建阶段：
   - 先复制各模块 `pom.xml`
   - 执行 `dependency:go-offline`
   - 再复制源码
   - 执行 `mvn -pl auction-admin -am package -DskipTests`
2. `eclipse-temurin:17-jre-alpine` 运行阶段：
   - 只复制最终 jar
   - 使用非 root 用户 `app`
   - 暴露 `8080`
   - 配置 `/actuator/health` 健康检查

## 3. 生产配置说明

`application-prod.yml` 不再连接 `localhost`，而是通过环境变量连接 Docker Compose 服务名：

| 配置 | 默认值 | 说明 |
|---|---|---|
| `MYSQL_HOST` | `mysql` | MySQL 服务名 |
| `REDIS_HOST` | `redis` | Redis 服务名 |
| `RABBITMQ_HOST` | `rabbitmq` | RabbitMQ 服务名 |
| `ES_HOST` | `elasticsearch` | Elasticsearch 服务名 |
| `JWT_SECRET` | 示例值 | 生产必须改强随机值 |
| `JAVA_OPTS` | `-Xms512m -Xmx1024m` | JVM 参数 |

## 4. 部署前校验

### 4.1 复制环境变量

```powershell
Copy-Item deploy\.env.example deploy\.env
```

生产环境必须修改：

- `MYSQL_ROOT_PASSWORD`
- `RABBITMQ_PASS`
- `MINIO_PASS`
- `JWT_SECRET`

### 4.2 校验 Compose 配置

不启动容器，只校验 YAML 展开结果：

```powershell
cd deploy
docker compose --env-file .env.example -f docker-compose.yml config
```

### 4.3 一键校验脚本

```powershell
powershell -ExecutionPolicy Bypass -File test/ops/day30-deploy-check.ps1
```

如果只想校验 Docker/Compose，不打 Maven 包：

```powershell
powershell -ExecutionPolicy Bypass -File test/ops/day30-deploy-check.ps1 -SkipMaven
```

## 5. 启动完整环境

确认机器内存足够后执行：

## 7. Nginx 反代说明

当前没有前端工程，因此 Nginx 只处理：

- `/api/**` → 后端 `backend:8080`
- `/actuator/health` → 后端健康检查
- `/uploads/**` → 后端上传文件访问
- `/` → 返回纯文本 `auction backend gateway ok`

后续接入前端后，可以把 `/` 改成静态前端目录或前端容器。

## 9. 常见问题

| 问题 | 原因 | 处理 |
|---|---|---|
| 后端容器连不上 MySQL | 仍使用 dev 配置或数据库未 healthy | 确认 `SPRING_PROFILES_ACTIVE=prod` 和 `MYSQL_HOST=mysql` |
| 后端健康检查失败 | 启动慢或依赖未就绪 | 查看 `docker logs auction-backend` |
| ES 启动失败 | 内存不足或数据目录权限问题 | 调低 JVM 或清理本地 data 后重试 |
| Nginx 502 | 后端未启动/未加入网络 | 检查 `auction-backend` 状态和 `auction-net` |
| Docker build 很慢 | Maven 依赖下载 | 首次构建正常，后续可利用 Docker layer cache |
